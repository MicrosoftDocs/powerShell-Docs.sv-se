# <a name="examples-of-comment-based-help"></a>Exempel på kommentarsbaserad hjälp

Det här avsnittet innehåller exempel som visar hur du använder kommentarbaserad hjälp för skript och funktioner.

## <a name="example-1-comment-based-help-for-a-function"></a>Exempel 1: Kommentarbaserad hjälp för en funktion

 Följande exempelfunktion innehåller kommentarbaserad hjälp.

```powershell
function Add-Extension
{
    param ([string]$Name,[string]$Extension = "txt")
    $name = $name + "." + $extension
    $name

    <#
        .SYNOPSIS
        Adds a file name extension to a supplied name.

        .DESCRIPTION
        Adds a file name extension to a supplied name.
        Takes any strings for the file name or extension.

        .PARAMETER Name
        Specifies the file name.

        .PARAMETER Extension
        Specifies the extension. "Txt" is the default.

        .INPUTS
        None. You cannot pipe objects to Add-Extension.

        .OUTPUTS
        System.String. Add-Extension returns a string with the extension or file name.

        .EXAMPLE
        C:\PS> extension -name "File"
        File.txt

        .EXAMPLE
        C:\PS> extension -name "File" -extension "doc"
        File.doc

        .EXAMPLE
        C:\PS> extension "File" "doc"
        File.doc

        .LINK
        Online version: http://www.fabrikam.com/extension.html

        .LINK
        Set-Item
    #>
}
```

Följande utdata visar resultatet av kommandot Get-Help som visar hjälpen för funktionen Lägg till tillägget.

```powershell
C:\PS> get-help add-extension -full
```

## <a name="example-2-comment-based-help-for-a-script"></a>Exempel 2: Kommentarbaserad hjälp för ett skript

Följande exempelfunktion innehåller kommentarbaserad hjälp.

Lägg märke till tomma rader mellan avslutande **#>** och `Param` instruktionen. I ett skript som inte har en `Param` -instruktionen, det måste finnas minst två tomma rader mellan sista kommentaren i hjälpavsnittet och första funktionsdeklarationen. Utan dessa tomma rader associerar Get-Help hjälpavsnittet med hjälp av funktionen i stället för skriptet.

```powershell
<#
  .SYNOPSIS
  Performs monthly data updates.

  .DESCRIPTION
  The Update-Month.ps1 script updates the registry with new data generated
  during the past month and generates a report.

  .PARAMETER InputPath
  Specifies the path to the CSV-based input file.

  .PARAMETER OutputPath
  Specifies the name and path for the CSV-based output file. By default,
  MonthlyUpdates.ps1 generates a name from the date and time it runs, and
  saves the output in the local directory.

  .INPUTS
  None. You cannot pipe objects to Update-Month.ps1.

  .OUTPUTS
  None. Update-Month.ps1 does not generate any output.

  .EXAMPLE
  C:\PS> .\Update-Month.ps1

  .EXAMPLE
  C:\PS> .\Update-Month.ps1 -inputpath C:\Data\January.csv

  .EXAMPLE
  C:\PS> .\Update-Month.ps1 -inputpath C:\Data\January.csv -outputPath C:\Reports\2009\January.csv
#>

param ([string]$InputPath, [string]$OutPutPath)

function Get-Data { }
```

Följande kommando hämtar skriptet hjälp. Eftersom skriptet inte är n måste en katalog som anges i miljövariabeln Path på kommandot Get-Help som hämtar skriptet hjälp med att ange skriptets sökväg.

```powershell
C:\PS> get-help c:\ps-test\update-month.ps1 -full
```


## <a name="example-3-parameter-descriptions-in-a-param-statement"></a>Exempel 3: Beskrivningar av parametern i en Param-instruktion

Det här exemplet visar hur du infogar parameterdescriptions i den `Param` av en funktion eller ett skript. Det här formatet är mest användbara när parametern är kort.

```powershell
function Add-Extension
{
    param
    (
        [string]
        # Specifies the file name.
        $name,

        [string]
        # Specifies the file name extension. "Txt" is the default.
        $extension = "txt"
    )
    $name = $name + "." + $extension
    $name

    <#
        .SYNOPSIS
        Adds a file name extension to a supplied name.
...
    #>
```

Resultatet är samma som resultat till exempel 1. Get-Help tolkar parameterbeskrivningar som om de har åtföljas av den `.Parameter` nyckelord.

## <a name="example-4--redirecting-to-an-xml-file"></a>Exempel 4:  Omdirigera till en XML-fil

Du kan skriva XML-baserade hjälpavsnitt för funktioner och skript. Även om det är enklare att implementera kommentarbaserad hjälp, krävs XML-baserade hjälp om du vill ha mer exakt kontroll över hjälpinnehållet eller om du översätta hjälpavsnitt till flera språk. I följande exempel visas de första raderna i skriptet uppdatering Month.ps1. Skriptet använder den `.ExternalHelp` nyckelord för att ange sökvägen till en XML-baserade hjälpavsnitt för skriptet.

```powershell
#  .ExternalHelp C:\MyScripts\Update-Month-Help.xml

    param ([string]$InputPath, [string]$OutPutPath)

    function Get-Data { }
```

I följande exempel visar användningen av den `.ExternalHelp` nyckelord i en funktion.

```powershell
function Add-Extension
{
    param ([string] $name, [string]$extension = "txt")
    $name = $name + "." + $extension
    $name

    # .ExternalHelp C:\ps-test\Add-Extension.xml
}
```

## <a name="example-5--redirecting-to-a-different-help-topic"></a>Exempel 5:  Omdirigera till ett annat hjälpavsnitt

Följande kod är ett utdrag från början av inbyggt `Help` funktion i Windows PowerShell, som visar en skärmbild av hjälptext i taget. Eftersom hjälpavsnittet för cmdleten Get-Help beskriver hjälpfunktionen, hjälpfunktionen använder den `.ForwardHelpTargetName` och `.ForwardHelpCategory` nyckelord för att omdirigera användaren till Get-Help cmdlet-hjälpavsnittet.

```powershell
function help
{

    <#
      .FORWARDHELPTARGETNAME Get-Help
      .FORWARDHELPCATEGORY Cmdlet
    #>
    [CmdletBinding(DefaultParameterSetName='AllUsersView')]
    param(
            [Parameter(Position=0, ValueFromPipelineByPropertyName=$true)]
            [System.String]
            ${Name},
    ...
```

Följande kommando använder den här funktionen. När en användare skriver ett Get-Help-kommando för hjälpfunktionen visas visar hjälpavsnittet för cmdleten Get-Help Get-Help.

```powershell
C:\PS> get-help help
```

```output
            NAME
                Get-Help

            SYNOPSIS
                Displays information about Windows PowerShell cmdlets and concepts.
            ...
```
